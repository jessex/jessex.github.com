tl;dr: My blog has been migrated from Posterous to my new site at jessex.ch!

The revolution will be televised The migration will be blogged about! After finally willing myself to put in the work, I have migrated my blog from Posterous to my personal site, jessex.ch. On the new site, one can expect to find a short auto-biographical summary on the front page, a catalog of my various projects, an action-packed blog and a dynamically feline-allocated 404 page. All future blog posts will be published directly on the new site. Thusly, this is my last (foreseen) post on the Posterous blog and the first post published directly on the new site.

Part of the desire to change formats was the Twitter acquisition of Posterous. It signaled that Posterous would eventually wind down support and shut its doors. I think it is better to be proactive now before I am left using whatever tools they provide at the last minute to migrate my data, if any. More critically, however, I simply wished to take control of my little piece of Internet real estate. The feeling of pushing posts to a hand-crafted site instead of a pre-packaged blogging platform, as nicely designed and implemented as it may be, is a warm and fuzzy one.

The new site is built using Tom Preston-Werner's Jekyll. It was easy to install and configure and get to building, nearly frictionless in the process. I find that Jekyll mostly stays out of your way, letting you spin up whatever you want while silently providing a minimal, light-weight framework to build off of. The Liquid template system is dynamic enough to let you get relatively fancy, but simple enough that a novice could get something built in a couple of hours. Configuration is a snap with YAML, particularly of post metadata via YAML front matter blocks on each post. And there's a post format for everyone, as long as everyone likes HTML, Markdown or Textile.

And to top it all off, the documentation is excellent.

To build the site, I used a modified Posterous migration script to grab all of my posts off of the old blog while maintaining smart permalinks and formatting. I used some basic grep'ing and regex'ing to modify permalinks, categories, tags and other metadata to my liking and then did some light, manual massaging of each post for polish. I then took control of the template engine and built on the design of the repository archive page, which used to be the only page on the site, to add in my blog archive. I laid out templates for the individual posts, the blog archive, the code archive and common elements thereof, and stylized the whole thing with some basic CSS. The actual process of building the site to its current state was only a couple of hours of work; time spent doing stylization was at least double of that as I repeatedly iterated on the design and worked out kinks which arose.

The domain name is registered through Switzerland's SWITCH, who primarily targets Swiss universities but is available to any one. The DNS is hosted by Namecheap's FreeDNS service. And the actual site is hosted at Github using their Github Pages service, which actually uses Jekyll on the back end to generate the sites. Being hosted by Github, jessex.github.com resolves to jessex.ch. I considered many hosting options before choosing Github, whom I went with largely for convenience's sake. Free, reliable, easy. That's precisely what I want in a basic, static, personal site.

There is still work to be done, however. I would like to add the ability to comment directly on blog posts, preferably with a system that takes the heavy lifting out of my hands such as Disqus. I would also like to add the ability to subscribe to blog updates, maybe utilizing something as simple as RSS. Code snippets and syntax highlighting are something to be explored, as well: at present, all snippets are hosted as Github Gists and embedded on the page via <script> tags. However, Jekyll provides syntax highlighting support via Pygments; doing code blocks directly within the post would cut out one more external dependency and give me more direct control over formatting.

And of course, the design, layout and structure are subject to change on a moment's notice.

But overall, I am quite pleased with how it has turned out thus far. I'm still pretty new to web development, particularly on the front end, and the tools provided by Jekyll helped to make my labors as easy as possible. Hopefully you will like the new site as much as I do! If you want to check out the source, it, like the site itself, is hosted on Github. Feel free to fork and improve!
